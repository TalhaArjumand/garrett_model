# Key Level Confluence Spec

## Purpose

This spec records the doctrine boundary between:

- key level as watch location
- confluence as qualification logic

It exists to prevent the repo from inventing a fake standalone
`active IRL` selection rule when the currently reviewed Garrett material does
not explicitly define one.

## Locked Predecessor Context

Already locked elsewhere in the repo:

- the universal model is the higher-level framework
- `IRL / ERL` define the current internal / external range-liquidity structure
- `IRL` is directly defined as `FVG`
- candle-formation logic is watched once price reaches a key level
- `ERL` functions as the target side in the current `internal to external`
  explanation

## Current Doctrine Boundary

No standalone explicit `active IRL` selection rule is currently locked from the
reviewed doctrine.

The current evidence supports a different framing:

- price reaches a candidate key level
- confluence then determines whether that location is actionable

This means key-level relevance should currently be treated as a qualification
problem, not as a single deterministic `IRL`-selection primitive.

## Confirmed Confluence Layer

The currently confirmed confluence layer is:

- candle formation at key level

At the current project state, this includes the already locked execution-side
grammar:

- `C2` closure
- `C3` support relative to `EQ(C2)`
- `C3` expansion confirmation
- `C4` continuation candidate relative to `EQ(C3)`

These candle rules do not select the key level by themselves.
They qualify whether price behavior at a key level is behaving correctly.

## Working Interpretation

The current working interpretation is:

1. identify the universal-model context
2. identify the relevant key-level area
3. wait for price to reach that key level
4. evaluate swing formation and later confluences there
5. if the formation qualifies, trade the expansion candle away from the level

So the current doctrine flow is:

- watch location first
- confluence second
- swing formation third
- expansion-candle execution fourth

Current reviewed execution clarification:

- key level is the location / context object
- swing formation is the confirmation / qualification object at that location
- the tradeable event is the subsequent expansion candle

In the currently reviewed candle grammar, this means:

- `C2` belongs to the swing-formation / closure side
- `C3` is the expansion candle
- `C4` is later continuation context, not the primary trade candle

## What Is Locked

Locked in the current repo doctrine:

- there is no standalone explicit `active IRL` selection rule currently locked
- key-level relevance is currently understood through confluence
- candle formation is one confirmed confluence layer at key level
- the current reviewed trade logic is:
  - key level -> swing formation -> expansion candle
- within the currently reviewed material, Garrett prefers the expansion candle
  rather than the raw reversal candle itself

## What Is Not Yet Locked

Not yet locked:

1. the other confluences Garrett uses beyond candle formation
2. whether those confluences are mandatory, optional, or contextual
3. whether Garrett later states a more explicit key-level qualification rule
4. whether confluence is hierarchical, additive, or narrative-dependent
5. how multiple candidate `IRL/FVG` levels should be ranked when more than one
   exists

## Implementation Status

Implementation status: not started.

No code should currently assume:

- a final `active IRL` selector
- a ranked FVG-selection algorithm
- a complete key-level qualification engine

until more direct doctrinal evidence is extracted.

## Non-goals

This spec does not yet define:

- final key-level scoring logic
- active `IRL` ranking
- multi-confluence weighting
- entry rules by itself
- target rules by itself
- whether this exact expansion-entry preference is universal across all later
  Garrett material
